# Progress
Тестовое задание (HTML, CSS, JS)

Сделано:
1. Дуга прогресса реализована с помощью svg-элементов.
2. Реализован программный интерфейс (API) управления параметрами прогресс-блока:  
 - setValue 
 - setAnimated 
 - setHidden

3. Для создания нового прогресс-блока написана функция, которая принимает на вход: 

 - DOM-элемент (который возьмёт на себя все функции progress), 
 - css-классы состояний, 
 - начальные параметры (опционально)

   `const progressBlock = createProgress(el, animatedClass, hiddenClass, initialParameters);`

    Функция возращает объект с методами управления параметрами.  
   Progress-компонент не зависит от панели управления и разметки страницы: он монтируется в переданный DOM-элемент и управляется через методы, возвращаемые функцией создания.

4. Состояния характерисуются css-классами:
 - Normal - по умолчанию  
 - Animated - 'progress-root--animated'  (при выключении тумблера progress возвращается к состоянию Normal)
 - Hidden - 'progress-root--hidden'  ("display: none" у svg-фигуры, контейнер оставляет место в разметке)

5. Чтобы компонент progress минимально зависел от вёрстки, были созданы css-классы, начинающиеся с префиксом progress-root, и сгруппированы в конце css-файла.
6. Для привязки прогресс-блока к панели управления создана функция initParametersPanel, которая принимает на вход элементы управления и прогресс-блок:

   `initParametersPanel (valueInput, toggleAnimation, toggleHidden, cbs)`  

7. Реализована валидация value-инпута последством разметки, но также компонент дополнительно защищён от некорректных значений.
8. Дизайн: элементы оформлены по предоставленному макету, инпуты стилизованы. Так как шрифт не был указан в ТЗ, выбран самостоятельно: Inter.
9. PixelPerfect: выполнен в браузере Chrome, есть небольшие расхождения по шрифту.
10. Кроссбраузерность: проверено с мобильного устройства (браузеры Chrome, Firefox, Samsung Internet).  
11. Реализованы состояния в css для инпутов (focus-visible, invalid) и чекбоксов (focus-visible, disabled) - дополнительно.

Субъективные комментарии:
* Панель управления реализована отдельной функцией, но при необходимости логика привязки контролов может быть встроена как метод экземпляра progress.
* Остановка анимации реализована через возврат к базовому состоянию Normal (начало дуги на 12 часов); более сложные UX-сценарии (пауза в текущем положении) возможны, но не реализованы в рамках прототипа.
