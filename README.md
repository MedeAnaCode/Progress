# Progress
Тестовое задание (HTML, CSS, JS)

Сделано:
1. Дуга прогресса реализована с помощью svg-элементов.
2. Созданы методы, относящиеся к экземпляру дуги, для переключения состояний (setValue, setAnimated, setHidden).
    Для создания нового прогресс-блока создана функция, которая принимает - DOM-элемент (который возьмёт на себя все функции progress), классы состояний и начальные параметры(опционально), и возращает объект с методами управления параметрами.  

   `const progressBlock = createProgress(el, animatedClass, hiddenClass, initialParameters);`  
3. Состояния характерисуются css-классами:
 - Normal - по умолчанию
 - Animated - 'progress-root--animated'
 - Hidden - 'progress-root--hidden'

4. Чтобы компонент progress минимально зависел от вёрстки, были созданы классы, начинающиеся на с префиксом progress-root и вынесены в конец css-файла.
4. Для привязки прогресс-блока к панели управления создана функция initParametersPanel, которая принимает на вход элементы управления и прогресс-блок:

   `initParametersPanel (valueInput, toggleAnimation, toggleHidden, cbs)`  

    * Субъективное замечание: честно говоря, хотелось сделать её одним из методов прогресс-блока (как-то логично), но подумала, что это может противоречить условию ("API не прибит к вёрстке").
5. Дизайн: элементы оформлены по референсу, стилизованы инпуты. Шрифт: Inter.
6. PixelPerfect: выполнен в браузере Chrome, есть расхождения по шрифту.
7. Кроссбраузерность: проверено с мобильного устройства, браузеры Chrome, Firefox, Samsung Internet.
8. Реализованы состояния в css для инпутов(focus-visible, invalid) и чекбоксов(focus-visible, disabled).
